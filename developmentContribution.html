Contributing

The source code for Billy is available on GitHub. We follow the gitFlow branching model.

Branches

  • master : Official Billy releases.
  • develop : Development branch. Features are developed from this branch on forks of the main repository

Fork and Pull Request

If you want to add a new feature or have a fix for some existing bug, you should first create your own fork(Check out GitHub's fork system). After that you can create your feature or bugfix in your fork. When your confident that your work is done, you should submit a Pull Request against the develop branch and await for someone to review your work.
